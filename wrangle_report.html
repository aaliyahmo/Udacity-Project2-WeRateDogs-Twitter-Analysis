This is a report stating the wrangling efforts

1. Data Gathering¶

Also known as acquire, store and retrieve from the above image.

All data was gathered from 3 sources:

  1. twitter archive of WeRateDogs which was provided by Udacity
  2. image prediction file, a url was provided to the file by Udacity
  3. json_text which was a extracted by querying Tweepy, twitter api, to get the favourite counts and retweet counts for each of the tweet_ids provided in twitter archive.

    A twitter developer account has to be created to use the twitter api. This was an additional step to retrieve codes needed to access the api to extract the data.

    These are 3 individual dataframes which will be manipulated for this project.

2. Accessing Data¶

This was a big part of the data wrangling.
I had to fully explore the data by looking at the data types, the wording of the headings, the value counts, check if anything overlapped, what needs to be dropped or combined etc.

I would dare say this is the fun part, it's like playing detective by looking for 'clues' or rather things that can be changed.
once the exploring/ accessing is done, a list was made of of all the Quality and Tidiness issues.

Quality issues concerns the accuracy and quality of the data.
Tidiness issues concerns the structure of the data.

I found 9 quality issues and 4 tidiness issues.

3. Cleaning Data¶

As the above picture states, data cleaning took the longest time. Everytime I thought I could move on and that its clean, I find something else.

I had to work on all the 9 issues and 4 tidy issues I found:

Quality issues:

  1. Dataset contains tweets that are replies and not the rated tweets.
    This was resolved by dropping the rows that have values in the "in reply to" columns in twitter archive dataframe.

  2. The timestamp column is in string format and not in datetime format.
    Changing the column to datetime by using to_datetime().

  3. There are only 4 types of values in the source column, this can be simplified.
    Using value_counts() and then changing the 4 values into appropriate names to be easily understood.

  4. Dataset contains tweets that are retweets and not the rated tweets.
    Remove rows that are retweets by dropping rows that have values in the "retweeted_status_id" column.

  5. Some tweets have a denominator that isn't 10.
    The goal is to focus on ratings out of 10, so all tweets that were not 10 were dropped. However, the tweet were first checked for correct ratings before dropping, if it was incorrect and the correct rating was in the text, the rating was changed.

  6. Many dog names are "none", "a" or "an"
    Using a for loop to try to find the names after keywords such as "Here is".

  7. The tweet_ID column should be string and not int format.
    Changing the tweet_id column using astype() as this column is not a column calculations would be run on.

  8. Tweets with missing images.
    expanded_url is the column that gives a link to a image. No url means that there is no image and the row would be dropped.

  9. Keeping tweets thats a max 15 numerator.
    Removing outliers by keeping 15 the max number for the numerator.


    Tidiness issues:

  10. Having one column for the 4 stages of dogs.
    The data started of with 4 columns, each column being a stage of dog (puppo, doggo, pupper and floofer), this is messy as usually theres only 1 type of dog stage per tweet. This would be simplified by creating 1 column called dog_stage.

  11. Remove purposeless (for our analysis) rows such as "in reply to" and "retweeted status".
    Dropping any unneccessary rows that will not be needed.

  12. Combine Json_tweets with df_archive.
    To compare the retweets and favourites with dog breeds, I had to join the dataframes as it would make it easier to analyze.

  13. Adding image_predictions into df_archive.
    To compare the dog breeds that were recognized from the tweet images, I had to merge the 2 dataframes but first taking the dog breed with the highest confidence level.
